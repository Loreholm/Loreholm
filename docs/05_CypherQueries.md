# Cypher Query Reference — loreholm (ArcadeDB)

This document defines the canonical Cypher queries used by loreholm’s service layer.

These queries are intentionally:
- **Boring**
- **Deterministic**
- **Inspectable**
- **POC-friendly**

They support the MCP tool contract defined in `docs/tool_schemas.md`.

---

## Labels & Relationship Types

### Labels
- `:Entity`
- `:Memory`
- `:Conversation`
- `:Message`

### Relationship types
- `:HAS_MESSAGE`
- `:MENTIONS`
- `:ABOUT`
- `:DERIVED_FROM`
- `:RELATED_TO`

---

## Property Conventions

### IDs
All node IDs are stored in an `id` property (opaque string generated by app).
Example:
- `Entity.id`
- `Memory.id`
- `Conversation.id`
- `Message.id`

### Normalization
Entities use a normalized key for matching:
- `Entity.name_norm` = lowercase(trim(name))
- `Entity.aliases_norm` = normalized aliases (array)

This is computed in the app, not in Cypher.

## Tool → Query Mapping

### 1) loreholm_upsert_entities

#### 1.1 Upsert Entity by normalized name (primary)

Parameters:

* `$entity_id`
* `$name`
* `$type`
* `$name_norm`
* `$aliases` (array)
* `$aliases_norm` (array)
* `$now` (ISO timestamp)

```cypher
MERGE (e:Entity {name_norm: $name_norm, type: $type})
ON CREATE SET
  e.id = $entity_id,
  e.name = $name,
  e.type = $type,
  e.aliases = $aliases,
  e.aliases_norm = $aliases_norm,
  e.created_at = $now,
  e.updated_at = $now
ON MATCH SET
  e.name = $name,
  e.aliases = coalesce(e.aliases, []) + $aliases,
  e.aliases_norm = coalesce(e.aliases_norm, []) + $aliases_norm,
  e.updated_at = $now
RETURN e.id AS entity_id, (e.created_at = $now) AS created;
```

> NOTE: This naïvely appends aliases. In the service layer, you should:
>
> * normalize + de-dupe aliases before passing
> * optionally cap alias list length for sanity

#### 1.2 Lookup by alias (secondary)

Used when name lookup misses, to prevent duplicates.
Parameters:

* `$alias_norm`
* `$type`

```cypher
MATCH (e:Entity {type: $type})
WHERE $alias_norm IN coalesce(e.aliases_norm, [])
RETURN e.id AS entity_id, e.name AS name, e.type AS type, e.aliases AS aliases
LIMIT 1;
```

If you find an alias match, then do a follow-up `SET` to add the new name/aliases:

```cypher
MATCH (e:Entity {id: $entity_id})
SET
  e.name = $name,
  e.updated_at = $now,
  e.aliases = coalesce(e.aliases, []) + $aliases,
  e.aliases_norm = coalesce(e.aliases_norm, []) + $aliases_norm
RETURN e.id AS entity_id;
```

### 2) loreholm_write_memory

This tool writes:

* `:Memory`
* optional `:Conversation` and `:Message` stubs (POC-friendly)
* links:

  * `(Memory)-[:ABOUT]->(Entity)`
  * `(Memory)-[:DERIVED_FROM]->(Message)`
  * `(Conversation)-[:HAS_MESSAGE]->(Message)`

#### 2.1 Upsert Conversation

Parameters:

* `$conversation_id`
* `$platform` (optional)
* `$started_at` (optional)
* `$now`

```cypher
MERGE (c:Conversation {id: $conversation_id})
ON CREATE SET
  c.platform = coalesce($platform, "unknown"),
  c.started_at = coalesce($started_at, $now),
  c.created_at = $now
RETURN c.id AS conversation_id;
```

#### 2.2 Upsert Message(s)

Parameters:

* `$message_id`
* `$conversation_id`
* `$role` (optional)
* `$text` (optional)
* `$timestamp` (optional)
* `$now`

```cypher
MATCH (c:Conversation {id: $conversation_id})
MERGE (msg:Message {id: $message_id})
ON CREATE SET
  msg.role = coalesce($role, "unknown"),
  msg.text = coalesce($text, ""),
  msg.timestamp = coalesce($timestamp, $now),
  msg.created_at = $now
MERGE (c)-[:HAS_MESSAGE]->(msg)
RETURN msg.id AS message_id;
```

> If you don't want to store message text, pass empty text.
> Provenance still exists via IDs.

#### 2.3 Create Memory and link to Entities + Message provenance

Parameters:

* `$memory_id`
* `$text`
* `$confidence`
* `$tags` (array)
* `$timestamp` (ISO)
* `$conversation_id`
* `$message_ids` (array)
* `$about_entity_ids` (array)

```cypher
CREATE (m:Memory {
  id: $memory_id,
  text: $text,
  confidence: $confidence,
  tags: $tags,
  timestamp: $timestamp,
  conversation_id: $conversation_id
})
WITH m
UNWIND coalesce($about_entity_ids, []) AS eid
MATCH (e:Entity {id: eid})
MERGE (m)-[:ABOUT]->(e)
WITH m
UNWIND coalesce($message_ids, []) AS mid
MATCH (msg:Message {id: mid})
MERGE (m)-[:DERIVED_FROM]->(msg)
RETURN m.id AS memory_id, m.timestamp AS timestamp;
```

### 3) loreholm_link_entities

Creates or updates a relationship.
Parameters:

* `$edge_id`
* `$from_entity_id`
* `$to_entity_id`
* `$relationship`
* `$confidence`
* `$reason`
* `$now`

```cypher
MATCH (a:Entity {id: $from_entity_id})
MATCH (b:Entity {id: $to_entity_id})
MERGE (a)-[r:RELATED_TO {relationship: $relationship}]->(b)
ON CREATE SET
  r.id = $edge_id,
  r.confidence = $confidence,
  r.reason = $reason,
  r.created_at = $now,
  r.updated_at = $now
ON MATCH SET
  r.confidence = $confidence,
  r.reason = $reason,
  r.updated_at = $now
RETURN r.id AS edge_id;
```

## Read Queries

### 4) loreholm_search (vector-first + fallback)

Canonical search strategy:

* generate embedding for query text
* run `vector_search.search` on `memory_embeddings`
* apply filters (time/tags/entity types)
* fallback to keyword path only if vector modules/index are unavailable

Parameters:

* `$query_text` (raw user query string)
* `$index_name` (default `memory_embeddings`)
* `$limit` (candidate count; service layer may over-fetch for filtering)
* `$since` (ISO timestamp or null)

#### 4.1 Vector search (primary path)

```cypher
CALL embeddings.text([$query_text]) YIELD embeddings
WITH embeddings[0] AS embedding
CALL vector_search.search($index_name, $limit, embedding)
YIELD distance, node, similarity
WITH node AS m, distance, similarity
WHERE ($since IS NULL OR m.timestamp >= $since)
OPTIONAL MATCH (m)-[:ABOUT]->(e:Entity)
WITH m, similarity, distance, collect({
  entity_id: e.id,
  name: e.name,
  type: e.type
}) AS entity_refs
RETURN
  m.id AS memory_id,
  m.text AS text,
  m.confidence AS confidence,
  m.timestamp AS timestamp,
  entity_refs AS entity_refs,
  m.conversation_id AS conversation_id,
  similarity AS similarity,
  distance AS distance
ORDER BY similarity DESC, m.timestamp DESC
LIMIT $limit;
```

#### 4.2 Lexical fallback: Search in Memory.text by token/phrase overlap

```cypher
MATCH (m:Memory)
WHERE ($since IS NULL OR m.timestamp >= $since)
WITH m, toLower(m.text) AS text_norm
WITH m, text_norm,
     size([term IN $terms WHERE text_norm CONTAINS term]) AS term_hits,
     size([phrase IN $phrases WHERE text_norm CONTAINS phrase]) AS phrase_hits
WHERE term_hits > 0 OR phrase_hits > 0
OPTIONAL MATCH (m)-[:ABOUT]->(e:Entity)
WITH m, term_hits, phrase_hits, collect({entity_id: e.id, name: e.name, type: e.type}) AS entity_refs
RETURN
  m.id AS memory_id,
  m.text AS text,
  m.confidence AS confidence,
  m.timestamp AS timestamp,
  entity_refs AS entity_refs,
  m.conversation_id AS conversation_id
ORDER BY phrase_hits DESC, term_hits DESC, m.timestamp DESC
LIMIT $limit;
```

#### 4.3 Lexical fallback: Search by Entity name/aliases → pull connected memories

```cypher
MATCH (e:Entity)
WITH e, toLower(e.name) AS name_norm, coalesce(e.aliases_norm, []) AS aliases_norm
WITH e,
     size([
       term IN $terms
       WHERE name_norm CONTAINS term
          OR any(alias IN aliases_norm WHERE alias CONTAINS term)
     ]) AS entity_hits
WHERE entity_hits > 0
MATCH (m:Memory)-[:ABOUT]->(e)
WHERE ($since IS NULL OR m.timestamp >= $since)
OPTIONAL MATCH (m)-[:ABOUT]->(e2:Entity)
WITH m, entity_hits, collect({entity_id: e2.id, name: e2.name, type: e2.type}) AS entity_refs
RETURN
  m.id AS memory_id,
  m.text AS text,
  m.confidence AS confidence,
  m.timestamp AS timestamp,
  entity_refs AS entity_refs,
  m.conversation_id AS conversation_id
ORDER BY entity_hits DESC, m.timestamp DESC
LIMIT $limit;
```

> Service layer should merge & de-dupe results across both queries.

### 5) loreholm_context (graph expansion)

Given seed entity IDs:

* fetch top memories about them
* fetch related entities (neighbors) within depth
* return compact context

Parameters:

* `$entity_ids` (array)
* `$limit` (int)

#### 5.1 Fetch memories about seed entities

```cypher
UNWIND $entity_ids AS eid
MATCH (e:Entity {id: eid})
OPTIONAL MATCH (m:Memory)-[:ABOUT]->(e)
WITH e, m
ORDER BY m.timestamp DESC
WITH e, collect(m)[..$limit] AS mems
UNWIND mems AS m
RETURN DISTINCT
  m.id AS memory_id,
  m.text AS text,
  m.confidence AS confidence,
  m.timestamp AS timestamp;
```

#### 5.2 Fetch neighbor entities (depth 1)

```cypher
UNWIND $entity_ids AS eid
MATCH (e:Entity {id: eid})
OPTIONAL MATCH (e)-[r:RELATED_TO]->(n:Entity)
RETURN DISTINCT
  n.id AS entity_id,
  n.name AS name,
  n.type AS type,
  r.relationship AS relationship,
  r.confidence AS confidence
LIMIT $limit;
```

#### 5.3 Depth 2 option (POC-safe)

If `depth >= 2`, do:

```cypher
UNWIND $entity_ids AS eid
MATCH (e:Entity {id: eid})
MATCH (e)-[:RELATED_TO]->(n1:Entity)-[:RELATED_TO]->(n2:Entity)
RETURN DISTINCT
  n2.id AS entity_id,
  n2.name AS name,
  n2.type AS type
LIMIT $limit;
```

> Service layer should keep depth limited by default and never explode the graph.

## Admin / Debug Queries

### 6) loreholm_recent

Parameters:

* `$limit`
* `$since` (optional)

```cypher
MATCH (m:Memory)
WHERE ($since IS NULL OR m.timestamp >= $since)
OPTIONAL MATCH (m)-[:ABOUT]->(e:Entity)
WITH m, collect({entity_id: e.id, name: e.name, type: e.type}) AS entity_refs
RETURN
  m.id AS memory_id,
  m.text AS text,
  m.confidence AS confidence,
  m.tags AS tags,
  m.timestamp AS timestamp,
  entity_refs AS entity_refs,
  m.conversation_id AS conversation_id
ORDER BY m.timestamp DESC
LIMIT $limit;
```

### 7) loreholm_stats

```cypher
MATCH (e:Entity) WITH count(e) AS entity_count
MATCH (m:Memory) WITH entity_count, count(m) AS memory_count
RETURN entity_count, memory_count;
```

Top entities by ABOUT degree:

```cypher
MATCH (e:Entity)<-[:ABOUT]-(m:Memory)
WITH e, count(m) AS mem_count
RETURN e.id AS entity_id, e.name AS name, e.type AS type, mem_count
ORDER BY mem_count DESC
LIMIT 20;
```

## Service-Layer Notes (important for correctness)

1. **De-dupe arrays in app code**

   * aliases, tags, message_ids, about_entity_ids

2. **Return compact context blocks**

   * Cypher returns raw rows; app formats for LLM consumption

3. **Prefer idempotent operations**

   * Upserts use `MERGE`
   * Writes use generated IDs and explicit creates

4. **Don’t let traversal explode**

   * Always enforce depth and limit
